# 🤝 Contributing

First of all, thank you for considering contributing to this project!

This platform is being built in public with the goal of documenting a real software engineering journey—from building projects to reaching a long-term goal of earning $1,000,000 through continuous learning and product development.

Whether you're fixing a typo, suggesting an improvement, or contributing code, every contribution is appreciated.

---

# Getting Started

1. Fork the repository.
2. Clone your fork.
3. Create a new branch.
4. Make your changes.
5. Test your changes.
6. Open a Pull Request.

---

# Branch Naming

Please use descriptive branch names.

Examples:

feature/project-page

feature/github-sync

fix/navbar-overflow

docs/update-readme

refactor/project-service

---

# Commit Convention

Use simple, descriptive commit messages.

Examples:

feat: add project card component

fix: resolve mobile navbar issue

docs: improve contributing guide

refactor: simplify timeline component

---

# Pull Requests

Please ensure:

- The feature works as expected.
- Code is clean and readable.
- Documentation is updated when needed.
- Screenshots are included for UI changes.
- PR focuses on a single feature or fix.

---

# Coding Principles

We value:

- Simplicity
- Readability
- Maintainability
- Performance
- Accessibility
- Consistency

Always prefer readable code over clever code.

---

# Reporting Bugs

When reporting bugs, include:

- Expected behavior
- Actual behavior
- Steps to reproduce
- Browser/OS (if applicable)
- Screenshots (if helpful)

---

# Feature Requests

Before opening a feature request:

- Check existing Issues.
- Explain the problem you're solving.
- Describe your proposed solution.
- Include mockups if applicable.

---

# Code Reviews

Every contribution is reviewed to ensure it aligns with the project's vision and quality standards.

Feedback is intended to improve the project, not criticize contributors.

---

# Thank You

Thanks for helping make this project better.

Let's build something meaningful together.
